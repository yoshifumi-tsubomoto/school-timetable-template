# 時間割アプリ 導入キット

中学校の時間割を、ブラウザ上で作成・変更・公開できるWebアプリです。 このリポジトリは、各学校が**自校専用のシステムとして導入する**ための一式です。

---

## まずはお試しください（体験版）

GitHub・Vercel・Supabaseの設定は一切不要です。下記のURLをクリックするだけで、今すぐ操作感を確認できます。

[**https://school-timetable-template.vercel.app/?demo=1**](https://school-timetable-template.vercel.app/?demo=1)

- ログイン不要・保存されません（架空データ「みどり野中学校」で動作）  
- 実際の学校のデータには一切触れません  
- 気に入ったら、下記の手順で自校専用のシステムとして導入してください

詳しい注意点は `docs/体験版の注意点.docx` を参照してください。

---

## 気に入ったら：自校用に保存するには

体験版はブラウザ上で動くだけで、何も保存されません。自校専用のシステムとして本当に使うには、 自校用のSupabase・Vercel・GitHubを用意する「初期設定」が必要です（下記「導入の流れ」参照）。

### 設定が難しい場合は、生成AIに手伝ってもらうのがおすすめです

GitHubやSupabaseの操作に不慣れでも心配いりません。ChatGPTやClaudeなどの生成AIに次のファイルを渡し、 「この手順書の通りに、自校用に導入したい」と伝えると、画面のスクリーンショットを見せながら ボタンの場所まで一つずつ教えてもらえます。

- 渡すファイル：`docs/正式版移行手順書.docx`  
- 進め方：手順で迷った画面のスクリーンショットをAIに見せながら、1ステップずつ確認して進める

⚠ **AIに見せるときの注意（プライバシー保護）**

- **送ってはいけないもの：** Supabaseの「データベースパスワード」、Google CloudのOAuth「クライアントシークレット」、 実際の生徒・保護者・教職員の氏名や個人情報が写った画面（先にモザイクなどで隠してください）  
- **送ってよいもの：** `docs/正式版移行手順書.docx`本体、作業中のGitHub/Vercel/Supabaseの操作画面（個人情報が写っていないもの）  
- 迷ったら、その情報が外部に漏れても困らないかどうかを基準に判断してください。

## はじめに：このアプリの大事な考え方

このアプリは、**学校ごとに完全に独立した1セット**を持つ仕組みです。 1つの学校 \= 専用のURL \+ 専用のデータ保管庫 \+ プログラムの自分用コピー。

- 自校のURL・データは自校だけのもので、他校とは完全に分かれています。  
- 自校で何を編集しても他校には影響せず、他校の操作が自校に及ぶこともありません。  
- 新しいバージョンを入れるかどうかも、各校が自分の判断で個別に決められます（自動では切り替わりません）。

⚠ **重要：** 同じURLを2校で共有することはできません。同じURL（＝同じ保管庫）を使うと、 2校が同じ1つの時間割を上書きし合ってしまいます。必ず学校ごとに1セットを用意してください。

## 導入に必要なもの

導入する学校ごとに、次の3つのサービスのアカウントを用意します（いずれも基本無料の範囲で始められます）。

| サービス | 役割 |
| :---- | :---- |
| **GitHub** | プログラム（ソースコード）を置く場所 |
| **Vercel** | コードを公開・配信し、URLを発行するホスティング |
| **Supabase** | ログイン認証と、時間割データの保存先（データベース） |

加えて、編集する管理者の **Googleアカウント**（ログインに使用）が必要です。

| ファイル | 内容 |

| `src/App.jsx` | アプリ本体（プログラム） |
| `src/config.js` | **接続先・学校名の雛形（空欄）**。自校の値を書き込んで使います |

| `CHANGELOG.md` | 変更履歴（どの版で何が変わったか） |

## 導入の流れ（ざっくり）

詳しい手順は `docs/正式版移行手順書.docx` にあります。大きな流れは次の6ステップです。

1. **Supabaseを準備** … 自校のプロジェクトを作り、必要なテーブルとアクセス制御（RLS）、Googleログインを設定する  
2. **コードを配置してデプロイ** … このリポジトリを自校用にコピーし、Vercelで公開する  
3. **管理者を登録** … 編集できる人のメールアドレスを登録する  
4. **学校データを投入** … 学級・教科・先生・担当などを「初期設定」から入力する  
5. **確認して公開** … 重複がないか確認し、閲覧URLを発行して配布する  
6. **運用開始**

### リポジトリのコピーの仕方

このリポジトリは **テンプレートリポジトリ** になっています。 GitHubで「**Use this template**（このテンプレートを使用）」を押すと、自校用のまっさらなコピーが作れます。

---

## ⛔ 最重要：config.js は必ず自校の値に書き換える

### なぜファイルが分かれているのか

このアプリは、共通のプログラム本体（`src/App.jsx`）と、学校ごとの設定（`src/config.js`）の 2つのファイルに分かれています。理由は次のとおりです。

- `App.jsx` は全校共通で、新しい版が出るたびに更新される部分です。  
- `config.js` は自校だけの接続先・学校名という、ごく小さな設定だけを持つ部分です。  
- 分けておくことで、新しい版に更新しても自校の設定は消えず、また複数校に配っても各校が独立して動きます。  
- `App.jsx` には秘密情報が含まれないので、生成AIに見せたり共有したりしても問題ありませんが、 `config.js` には自校固有の値が入るため、取り扱いには注意してください（上記「AIに見せるときの注意」参照）。

`src/config.js` には、**どの保管庫（Supabase）につなぐか**と**学校名**が書かれています。 このキットの `config.js` は**空欄の雛形**です。導入時に、必ず**自校の値**に書き換えてください。

// src/config.js

export const SUPABASE\_URL \= "https://〇〇.supabase.co";   // 自校のProject URL

export const SUPABASE\_KEY \= "eyJ...";                      // 自校のanon（公開）キー

export const SCHOOL\_NAME  \= "〇〇中学校";                   // 自校の学校名（初期表示用）

- 他校から受け取った `config.js`（他校の値入り）を**そのまま使わない**でください。他校のデータにつながってしまいます。  
- `config.js` は `src/App.jsx` と**同じ `src` フォルダ**に置きます。無いとデプロイが失敗します。  
- anonキーはブラウザに配信される公開用のキーなので、リポジトリに置いて問題ありません（データの保護はSupabase側のRLSで行います）。

## バージョンアップ（新しい版への更新）

### 新しい版が出たことに気づくには

このリポジトリでは、新しい版を出すたびに「Releases」にバージョンを公開しています。 リポジトリ右上の「Watch」→「Custom」→「Releases」にチェックを入れておくと、 新しい版が出るたびにメールで通知が届きます（コミットのたびに通知は来ません）。

新しい版が配布されたら、更新するのは基本的に **`src/App.jsx` だけ**です。

1. GitHubで `src/App.jsx` を開き、編集に入る  
2. **全選択（Ctrl+A）→ 新しいコードを全置換**で貼り付ける  
3. コミットすると、Vercelが自動で再公開する

`config.js`（自校の接続先・学校名）は触らないので、更新しても消えません。

⛔ **貼り付け事故に注意：** 一部だけ貼ると先頭・末尾が欠けてビルドエラーになります。 必ず**全選択→全置換**でファイル全体を入れ替えてください。

⛔ **重要：** 新しい版で `config.js` から読み込む項目が増える場合（リリースノートに記載）は、 必ず①`config.js`の更新→②`App.jsx`の更新の順で、同じブランチにコミットしてからPreviewを確認してください。

更新を入れるかどうかは各校の判断です。何が変わったかは `CHANGELOG.md` を確認してください。

## 改善要望・不具合の連絡

「こうだと使いやすい」「ここがうまく動かない」といった声を歓迎します。 ただし、このアプリは大きな1ファイル構成のため、**コードを直接書き換えて送るのではなく、要望を文章でお寄せください**。 本家でまとめて実装し、新しい版として配布します。

- **GitHubが使える場合：** 配布元の本家リポジトリのIssuesから投稿してください（テンプレートがあります）。 自校用にコピーした後は、コピー先ではなく必ずこちらの本家にご投稿ください。 → [https://github.com/yoshifumi-tsubomoto/school-timetable-template/issues/new/choose](https://github.com/yoshifumi-tsubomoto/school-timetable-template/issues/new/choose)  
- **GitHubを使わない場合：** 配布元が用意する改善要望フォームからご連絡ください

### 自校での改変について

自校の実態に合わせて、ソースコード（`src/App.jsx`など）を生成AIに渡し、改変することを許可します。 基本の機能はすでにできているので、ゼロから作るよりずっと早く自校専用の時間割アプリに仕上げられます。 `App.jsx` には生徒等の個人情報は含まれていない（データはSupabase側にあります）ので、AIに渡すこと自体は問題ありません。

- おすすめの改変があれば、ぜひ本家にもお知らせください。採用を検討し、次の版に取り込むことがあります。 お知らせの際は、**どの版（バージョン番号）をもとにしたか**・**改変日時**・**改変内容**を添えてください。  
- 元のプログラムの著作権は坪本吉史（開発者）に帰属します。改変した場合も、この著作権表示は残してください。  
- ⚠ **改変すると、以後の「バージョンアップ（全選択→全置換）」の手順がそのままは使えなくなります。** 新しい公式版をそのまま貼り付けると、自校の改変部分が失われます。公式の更新を取り込みたい場合は、 改変箇所を控えておき、新しい版に手動で反映する（または生成AIに両方の版を見せて差分だけ移植してもらう）必要があります。  
- 改変した版の動作について、本家（開発者）はサポート・保証を行いません。
