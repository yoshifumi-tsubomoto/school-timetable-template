時間割アプリ 導入キット
中学校の時間割を、ブラウザ上で作成・変更・公開できるWebアプリです。 このリポジトリは、各学校が自校専用のシステムとして導入するための一式です。

まずはお試しください（体験版）
GitHub・Vercel・Supabaseの設定は一切不要です。下記のURLをクリックするだけで、今すぐ操作感を確認できます。
https://school-timetable-template.vercel.app/?demo=1
ログイン不要・保存されません（架空データ「みどり野中学校」で動作）
実際の学校のデータには一切触れません
気に入ったら、下記の手順で自校専用のシステムとして導入してください
詳しい注意点は docs/体験版の注意点.docx を参照してください。

はじめに：このアプリの大事な考え方
このアプリは、学校ごとに完全に独立した1セットを持つ仕組みです。 1つの学校 = 専用のURL + 専用のデータ保管庫 + プログラムの自分用コピー。
自校のURL・データは自校だけのもので、他校とは完全に分かれています。
自校で何を編集しても他校には影響せず、他校の操作が自校に及ぶこともありません。
新しいバージョンを入れるかどうかも、各校が自分の判断で個別に決められます（自動では切り替わりません）。
⚠ 重要： 同じURLを2校で共有することはできません。同じURL（＝同じ保管庫）を使うと、 2校が同じ1つの時間割を上書きし合ってしまいます。必ず学校ごとに1セットを用意してください。

導入の流れ（ざっくり）
詳しい手順は docs/正式版移行手順書.docx にあります。大きな流れは次の6ステップです。
Supabaseを準備 … 自校のプロジェクトを作り、必要なテーブルとアクセス制御（RLS）、Googleログインを設定する
コードを配置してデプロイ … このリポジトリを自校用にコピーし、Vercelで公開する
管理者を登録 … 編集できる人のメールアドレスを登録する
学校データを投入 … 学級・教科・先生・担当などを「初期設定」から入力する
確認して公開 … 重複がないか確認し、閲覧URLを発行して配布する
運用開始
リポジトリのコピーの仕方
このリポジトリは テンプレートリポジトリ になっています。 GitHubで「Use this template（このテンプレートを使用）」を押すと、自校用のまっさらなコピーが作れます。

⛔ 最重要：config.js は必ず自校の値に書き換える
src/config.js には、どの保管庫（Supabase）につなぐかと学校名が書かれています。 このキットの config.js は空欄の雛形です。導入時に、必ず自校の値に書き換えてください。
// src/config.js
export const SUPABASE_URL = "https://〇〇.supabase.co";   // 自校のProject URL
export const SUPABASE_KEY = "eyJ...";                      // 自校のanon（公開）キー
export const SCHOOL_NAME  = "〇〇中学校";                   // 自校の学校名（初期表示用）
他校から受け取った config.js（他校の値入り）をそのまま使わないでください。他校のデータにつながってしまいます。
config.js は src/App.jsx と同じ src フォルダに置きます。無いとデプロイが失敗します。
anonキーはブラウザに配信される公開用のキーなので、リポジトリに置いて問題ありません（データの保護はSupabase側のRLSで行います）。

バージョンアップ（新しい版への更新）
新しい版が配布されたら、更新するのは基本的に src/App.jsx だけです。
GitHubで src/App.jsx を開き、編集に入る
全選択（Ctrl+A）→ 新しいコードを全置換で貼り付ける
コミットすると、Vercelが自動で再公開する
config.js（自校の接続先・学校名）は触らないので、更新しても消えません。
⛔ 貼り付け事故に注意： 一部だけ貼ると先頭・末尾が欠けてビルドエラーになります。 必ず全選択→全置換でファイル全体を入れ替えてください。
⛔ 重要： 新しい版で config.js から読み込む項目が増える場合（リリースノートに記載）は、 必ず①config.jsの更新→②App.jsxの更新の順で、同じブランチにコミットしてからPreviewを確認してください。
更新を入れるかどうかは各校の判断です。何が変わったかは CHANGELOG.md を確認してください。

改善要望・不具合の連絡
「こうだと使いやすい」「ここがうまく動かない」といった声を歓迎します。 ただし、このアプリは大きな1ファイル構成のため、コードを直接書き換えて送るのではなく、要望を文章でお寄せください。 本家でまとめて実装し、新しい版として配布します。
GitHubが使える場合： このリポジトリの「Issues」から投稿（テンプレートがあります）
GitHubを使わない場合： 配布元が用意する改善要望フォームからご連絡ください

利用について
このアプリは学校での時間割運用のために提供しています。
導入・運用は各校の責任で行ってください（データの保護・バックアップを含む）。
改変は各校の自由ですが、改変した版についてのサポートは行えません。
※ 利用条件の正式な文面は、配布元の方針に従ってください。
