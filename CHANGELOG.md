# 変更履歴（CHANGELOG）

このアプリの版ごとの変更点をまとめます。新しい版を入れるかどうかの判断材料にしてください。
新しい版が上にくるように、上へ追記していきます。

書き方のめやす（各版で、当てはまるものだけ書けばOKです）：

- **追加** … 新しくできるようになったこと
- **改善** … これまでの動きが良くなったこと
- **修正** … 不具合を直したこと
- **注意** … 更新時に気をつけること（あれば）

各校での更新は、`src/App.jsx` を全選択→全置換で貼り替えるだけです。
ヘッダーやセルの描画など構造に関わる更新は、本番に直接入れず、まずブランチを作ってプレビューURLで確認してください。

## [8_7_4] － 2026-06-25

学級別ビューで重複（同じ学級・同じ時限に2つの授業）が分かりにくかった点を改善しました。

- **改善：** 重複しているコマに、太い赤枠と右上の「⚠」バッジを表示するようにした。これまで授業者別ビューでしか目立たなかった重複が、学級別ビューでもひと目で分かる。
- **修正：** 学級別ビューで、合同授業やA/B週分割のコマなど、一部の重複が表示されない取りこぼしがあったのを修正（base と changes の両方を見て判定するようにした）。
- **追加：** ヘッダーの「⚠ 重複 ◯件 ▸」がボタンになり、押すたびに重複している箇所へ1つずつ移動し、該当のコマを数秒間点滅して示すようにした。どの画面からでも、押していくだけで全部の重複を順番に確認できる。
- **改善：** 重複件数の数え方を「重複している箇所の数」に変更（同じコマに2つ重なっていても1箇所と数える）。ボタンを押す回数と数字が一致して分かりやすくなった。

（※ 開発途中で「マウスを乗せると自動で移動」する動きも試したが、紛らわしいため最終的に外し、ボタンで移動する方式に統一しました。）

## [8_6_0] － 2026-06-24

- **改善：** ヘッダーのメニューを整理し、マウスを乗せると開くドロップダウン方式にした。項目が増えて横に長くなっていたのを3つのメニューに集約。
  - 「生徒ビュー」「教員ビュー」→「ビュー切り替え」にまとめた。
  - 「集計」「URL管理」→「その他」にまとめた（体験版では URL管理 は非表示のため「集計」のみ）。
  - 「会議管理」→「初期設定」メニューの中に移動した。

---

## [8_5_0] － 2026-06-23

- **改善：** バージョン番号をヘッダーの学校名のとなりに表示するようにした（例：v8_5_0）。どの版が動いているか画面で確認できる。
- **注意：** 接続先（SUPABASE_URL / SUPABASE_KEY）を `config.js` に分離する方式を一度試したが、本番ビルドで不具合が出たため見送り。現在は本体（App.jsx）に直接書く方式で運用中。多校展開の際に、プレビューURLで検証してから改めて導入予定。
- **追加：** 導入キット（README・変更履歴・要望テンプレート）を整備。
